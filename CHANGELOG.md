## 🔧 修复的问题

### 1. ⚠️ **严重bug：血糖单位错误** (已修复)

**问题描述**：
- 输入界面显示 **mg/dL**，但代码判断阈值用的是 **mmol/L**
- 7.8 mmol/L ≈ 140 mg/dL
- 这导致所有血糖 >140 mg/dL 的患者被误判为高风险

**修复方案**：
```javascript
// 添加单位转换函数
function mgDlToMmolL(mgDl) {
    return mgDl / 18;  // 1 mmol/L = 18 mg/dL
}

// 在计算中使用转换后的值
const glucoseMmolL = mgDlToMmolL(glucoseMgDl);

// 修正后的阈值
if (glucoseMmolL > 11.1) riskScore += 0.10;  // >200 mg/dL
else if (glucoseMmolL > 7.8) riskScore += 0.05;  // >140 mg/dL
else if (glucoseMmolL > 5.6) riskScore += 0.02;  // >100 mg/dL
```

### 2. 🎨 **仪表盘起始位置错误** (已修复)

**问题描述**：
- 仪表盘指针没有从0%（最左侧）开始
- 0%位置应该是 Math.PI（左侧），100%位置是 0（右侧）

**修复方案**：
```javascript
// 正确的角度计算
// 0% = Math.PI (左侧), 100% = 0 (右侧)
const needleAngle = Math.PI * (1 - probability);

// 绘制三段颜色区域
// Green: 0-10%, Yellow: 10-20%, Red: 20-100%
```

### 3. 📊 **风险阈值缺乏文献支持** (已调整)

**原设置**：
- 低风险: <15%
- 中风险: 15-30%
- 高风险: >30%

**问题**：
- 15%阈值过于保守，导致大多数患者被分入中高风险
- 难以产生"正常值"的低风险预测

**新设置**（基于文献）：
- **低风险**: <10%
- **中风险**: 10-20%
- **高风险**: 20-30%
- **极高风险**: >30%

## 📚 文献依据

### 风险分层阈值来源

| 来源 | 低风险 | 中风险 | 高风险 | 极高风险 |
|------|--------|--------|--------|----------|
| **RISK-PCI Score** | ≤3% | 6-12% | 13-18% | >29% |
| **MBF+GLS Model** | <30% | 30-40% | >40% | - |
| **GRACE Score** | <3% | 3-6% | >6% | - |
| **本网站采用** | <10% | 10-20% | 20-30% | >30% |

### 关键参考文献

1. **RISK-PCI Score**
   - 低风险 (0-2.5分): 1.9-3.2% MACE率
   - 中风险 (3-4.5分): 5.9-12.5% MACE率
   - 高风险 (5-6.5分): 13.3-18.2% MACE率
   - 极高风险 (≥7分): 29.4-39.4% MACE率

2. **MBF + GLS Model** (2023, Int J Cardiovasc Imaging)
   - 最佳阈值: 40%
   - 分层: 30% 和 70% 分界
   - AUC: 0.95 (敏感性 84%, 特异性 94%)

3. **临床决策曲线分析**
   - 多数模型在阈值概率 0.1-0.78 范围内有净临床收益
   - 0.14-0.66 是常用的有效范围

**详细文献信息请查看**: `LITERATURE.md`

## ➕ 新增功能

### 1. 🧮 BMI自动计算器
- 输入身高(cm)和体重(kg)自动计算BMI
- BMI显示为只读字段，避免手动输入错误
- 计算公式: BMI = weight (kg) / (height (m))²
- 示例: 身高170cm, 体重71kg → BMI = 71 / (1.7)² = 24.6

### 2. 🌐 中英文切换
- 点击右上角按钮可在中文/英文间切换
- 所有界面元素、风险解读、推荐方案都支持双语

### 3. 📈 改进的风险分层
新增"极高风险"级别（>30%），用于识别需要紧急干预的患者

### 4. 🔧 校准的计算公式
- 基础风险从 10% 调整为 15%（STEMI患者基线风险）
- 各风险因子的权重重新校准
- Logistic函数的参数调整，使输出范围合理（2%-85%）
